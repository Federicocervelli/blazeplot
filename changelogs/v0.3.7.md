# BlazePlot v0.3.7

## Changes

- Made chart rendering on-demand by default so static charts render once after `start()` and then sleep until chart-owned state changes.
- Added `start({ renderLoop: "continuous" })` for benchmarks, external animation loops, and other cases that need every-frame rendering.
- Made repeated `chart.start()` calls idempotent and ensured `stop()` cancels scheduled rendering.
- Added a website render-loop preview comparing on-demand and continuous rendering with interaction controls.
- Updated benchmark and interaction coverage for the new render scheduling behavior.
- Documented the manual Cloudflare preview workflow.
