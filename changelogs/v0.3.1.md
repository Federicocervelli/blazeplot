# BlazePlot v0.3.1

This patch moves releases to the branch workflow used by the repo going forward.

## Changes

- Added a CI workflow for pull requests to `main` and `development`, plus pushes to `development`.
- Changed npm/GitHub releases to run from merges to `main` when `package.json` contains an unpublished version.
- Switched the Pages preview deployment from `master` to `main`.
- Replaced the old push/tag release script with local version-prep commands that create a release PR instead of publishing directly.
