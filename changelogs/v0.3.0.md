# BlazePlot v0.3.0

This release turns BlazePlot from a fast line-chart prototype into a broader real-time charting toolkit: richer built-in plugins, better time-series ergonomics, React and linked-chart helpers, and a cleaner high-performance streaming data path.

## Highlights

- Added first-class interaction plugins for brush selection, crosshair/ruler measurements, hover markers, and navigator overview windows.
- Added linked chart layout helpers for synchronized multi-chart dashboards.
- Added a React wrapper subpath (`blazeplot/react`).
- Added chart titles, axis titles, configurable axis scale ticks, time-axis formatting, and improved right-axis interaction support.
- Added the public `UniformRingBuffer` dataset for high-throughput uniformly spaced streams.
- Added y-only appends via `SeriesStore.appendY()` for implicit-X datasets.
- Made `capacity` optional when a dataset is supplied to a series.
- Improved hover responsiveness by coalescing pointer-move hover refreshes to animation frames.

## Data and performance

- New `UniformRingBuffer` stores only Y values and derives X from `xStart + index * xStep`, with built-in accelerated min/max extraction for dense rendering.
- New `YAppendableDataset` contract supports efficient y-only streaming APIs.
- `SeriesStore.markDirty()` gives advanced custom datasets an explicit invalidation path.
- Dataset-backed series no longer need to repeat `capacity` in their config.
- Scatter and hover behavior received several fixes and optimizations around point lookup, crosshair data, and DevTools-sensitive pointer-move paths.

## Plugins and UI

- New brush selection plugin.
- New crosshair ruler mode with configurable/readable measurements.
- New navigator overview plugin with edge resizing and plot-aligned layout.
- New linked-charts helper.
- New React wrapper entry point.
- Added richer chart event subscriptions, including pointer and selection events.
- Added chart/axis title support and time tick formatting.

## Preview and examples

- Added a feature showcase page.
- Reworked the main preview to use public library data primitives instead of preview-only accelerated datasets.
- Made preview sample rate and visible sample span configurable.
- Fixed preview live-stream timestamp behavior so increasing sample rate increases granularity without pushing data into the future.
- Simplified preview performance stats to focus on the useful runtime metrics.

## Fixes and polish

- Improved navigator theming, layout reservation, and edge resize behavior.
- Improved crosshair layering, active-series filtering, label defaults, and readout placement.
- Improved feature-preview sizing, tooltip formatting, and box zoom behavior.
- Added roadmap/benchmark documentation updates.
